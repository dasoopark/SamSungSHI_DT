서버 - 서비스를 제공하는 측( 하드웨어, 소프트웨어, ) 서버용 컴퓨터, json-server, tomcat, iis, 카카오 서비스를
클라이언트 - 서비스를 제공받는 측 ( 하드웨어, 소프트웨어 ), PC, 브라우저

프로토콜 - 네트워크를 통해서 데이터를 주고 받으려면, 서로 약속이 이썽야 하는데 약속을 프로토콜 이라고 한다.
        웹 : http, https    http://localhost:3000
            http: 웹이라는 방식으로 데이터를 주고 받는다. 프로토콜
           localhost : 서버랑 클라이언트가 데이터를 주고 받는다. 프로토콜 아이피, mac address(lan 카드 주소), port
           아이피는 논리적인 주소(특정 컴퓨터에 붙박이가 아니고 바뀔수 있다)
           mac address - 각 컴퓨터마다 통신이 가능하려면 lan카드가 설치되어야 한다.
           각 랜카드를 식별하기 위한 값으로 전세계 유일성을 보장한다.
           cmd -> ipconfig/all 물리적주소(mac-address)

           localhost, 127.0.0.1 - loop back 주소, 본인컴퓨터

           ip알고 
           mac address 도 알고
           A라는 컴퓨터 (브라우저)                  B컴퓨터(PORT번호)
           A ===========================================> B
           http  -  80      http://google.com:80
           json-server라는 프로그램은 포트번호 3000 를 선택 http://127.0.0.1:3000 으로 접근
           mysql - 3306
           oracle - 1521
           mssql - 1443
           ftp - 23         : 파일을 주고 받기 위한 프로토콜
           sftp, ssh - 22   : telnet 업그레이드, ssh 서버설치  sftp라는 프로토콜도 작동(파일전송)
           telnet - 21      : 원격으로 서버에 접근하기 위한 프로토콜, 서버로 한글자씩 보내서 단어가 치다가 잘못되면 첨부터 다시
           SNMP             : PC방에서 다른컴퓨터 제어


아이피  - 자동으로 (자동으로 할당)
          서버로 작동시키려면 고정아이피가 필요, 돈내고 고정아이피 서비스를 받던가 
          AWS나 네이버에 배포하면 AWS(아마존이 제공하는 클라우드 서비스)가 도메인을 제공해준다.
